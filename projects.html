Seattle Fire Dispatch
Fullstack
WIP

Note: This project is a work in progress and is not complete. In the close future I hope to host the application on the web for others to use!

Goal

I have always enjoyed seeing what the Seattle Fire Department is doing through out the day. However, while I can view the department's activity on their website I always wondered the following:

  • Where is that incident in relation to where I am?
  • What does do the emergency code mean?
  • How and where can I learn more about a large event?
  • What do the vehicle codes mean and what do the vehicles look like?
  • How common are these types of incidents?
  • How long does it take for the department to arrive to an incident and how long are they there?
With this project I hope to create an easy way for people to learn and share info about incidents in the Seattle area.

Another large driving force for this project is building my skills as a fullstack developer and learning more about starting a business.

Notakto
Java
Github

Notakto is a variant of tic-tac-toe which can be played across one or several boards. Both players use the same piece, an "X", with the goal of forcing the opponent to make a three-in-a-row of Xs. The game is traditional played on a 3x3 board, however any NxM board is possible where N,M >= 3. The game has a distinct advantage over tic-tac-toe as there will always be a winner, however the winner is often determined by who went first and the size of the board.

The game includes different choices of board size, a player verse player mode, and three different levels of AI to play against. The AI play the game as following:

  • Easy: Make a random move into an unoccupied square.
  • Medium: Make a random move into an unoccupied square which will not result in an immediate loss.
  • Hard: Make an optimal move into an unoccupied square which will not result in an immediate loss and maximizes the chance of winning next turn. Look ahead by one turn.

Voronoi Diagram Art
Math
Art
Github
What are Voronoi Diagrams?

A voronoi diagram is a partitioning of a plane with n generating points into convex polygons such that each polygon contains exactly one generating point and every point in a given polygon is closer to its generating point than any other point. They are also known as Voronoi tessellation, Voronoi decomposition, Voronoi Partition, or Dirichlet tessellation.

Creating Art

Using a Voronoi Diagram we can create mosaic or plainting like repersentations of an original image. To do this we use the following algorithm:

  • Given an image I, generate a random set of n points P which lies within the perimeter of I.
  • Generate a Voronoi Diagram V from the set of points P. Let v_i denote a polygon in V.
  • Color each polygon v_i using the pixel valaue from I located at the center of v_i.
  • If a polygon v_i center is not located on I, color v_i using the pixel value from the closest point on I to the center of v_i.

While using the algorithm the final image can vastly differ depending on your set of points P. For instance, if you generate your set P using a Gaussian distribution, increasing the amount of points n improves the clarity of the final image. While using a skewed distribution to create P can make a sense of depth or focus to the final image.

Vapor-to-Particle Reactions
Math
Code
Github

A toolset created in Matlab to aid in empirical data collection of vapor-to-particle reactions for a chemistry lab at Colorado State University. The tool set consists of four main programs these being:

  • ColorSpec - Analysis the RGB values of an image or video in a given region. The user can specify to break the region into vertical or horizontal slices for further analysis.
  • LineSpec - Analyzes the color intensity and light intensity of an image or video on a given line.
  • Crystal Analysis - Maps regions of a particle or material to show the underlying crystal structure through the use of a 2d fourier transform. If the region consists of only noise or no substantial phase cannot be found, the region is further broken up regions and analysed.
  • TornadoPulse - Analyzes the changing volume and perimeter of a particle cloud. This allows the user to measure the volume of the vortex rings leaving the vapor-to-particle site as well as the frequency.
Color Spec

The user provides a photo or video to be analysed using a gui. Once the media is loaded into memory the user then selects a region of the media. The RGB values from the media is then analyzed as a histogram and then vertical cross sections.

Line Spec

The user loads in a photo or video and draws a pixel thin line on the media. The pixels across the user's line are then analyzed per a frame and produces live data.

Crystal Analysis

The user provides an image from Digital Microscope (dm3). The image is recursively broken up into regions to fit within the edges of the detected particle. Each region then undergoes a 2d fourier transform to detect the orientation of crystalline structure with in the region. Regions are then grouped together using a K clustering algorithm and colored according to its grouping.

Tornado Pulse

This application looks as the light intensity from a video and identifies a blob using standard deviations. The area, center and perimeter of the blob are recorded over time, blobs whose center is outside of the blue bounding box are ignored. The tool allows the user to measure the rate at which the aerosol leaves the droplet.

Grind, Level Design
CSGO
Level Design

A Counter Strike: Global Offensive bomb defusal map set in Namibia mine. Featuring a highly competitive layout focused around map control, and 100% custom environment art.

History

I original started the de_grind as a passion project to reinvent de_dust, a classic Counter Strike map which has fallen out of popularity. While very little of de_dust exists in the final version of de_grind the competitive feel and the art style, albeit more industrial, carried over into de_grind.

Over the course of 2 years the project took a lot of different directions, styles, and concepts before reaching its finished state. Early on the map went through many themes; an sandy North Africa theme, then a small coastal Italian village, and then through a modern condo theme. During this time I grew rapidly as a level designer and more proficient with the toolset.

I eventually joined Mapcore, a level design community, where I meet Jan Huygelen. Together we worked on polishing the gameplay and trying to pin down a theme. Unfortunately, before we could pin down a theme and finish the map Jan got a offer to work on a high profile project, de_mirage2 later called de_thrill. While working with Jan, I learned the depths of the source engine and developed my skill set as an artist.

As I was committed to finish the project and did not have the skill set to finish the project on my own, I sought after a new team member. I eventually recruited Konstantin Loffler, another level designer, to finalize the gameplay of the map. Together we extensively playtested the map and settled on the final theme, a modern mining facility.

We then recruited Cory de la Torre to act as our lead Environment Artist. With his help we finished the art and shipped with 100% self generated art content.
